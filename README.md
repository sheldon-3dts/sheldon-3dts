## Reading list

A small collection of papers/articles/books that I think are useful & relevant for software engineers to read.

These are just some of the ones I can remember, please feel free to submit more links/suggestions to this doc!

### Short articles
- [grugbain](https://grugbrain.dev/) - meme-ified complexity=bad article
- [cognitive load](https://github.com/zakirullin/cognitive-load) - tldr; on cognitive load and why it's bad
- [How to read a paper](https://web.stanford.edu/class/ee384m/Handouts/HowtoReadPaper.pdf) - An improved way to read through academic papers/articles
- [Towards an API for the real numbers](https://dl.acm.org/doi/pdf/10.1145/3385412.3386037) - links to the 'how hard could a calculator be'
- [Framework rant](https://minds.md/benji/frameworks)
